CSP:
✅ default-src 'self' → Blocks unauthorized external scripts unless explicitly allowed.
✅ script-src 'self' https://challenges.cloudflare.com → Prevents external scripts except for Cloudflare Turnstile.
✅ connect-src 'self' https://api.coingecko.com → Allows only CoinGecko API requests (prevents unauthorized external fetches).
✅ style-src 'self' 'unsafe-inline' https://cdnjs.cloudflare.com https://fonts.googleapis.com → Prevents unwanted inline styles, except necessary ones.
✅ font-src 'self' https://fonts.gstatic.com https://cdnjs.cloudflare.com → Prevents third-party font injection attacks.
✅ img-src 'self' data: https://coin-images.coingecko.com → Blocks unauthorized external images, only allowing CoinGecko.
✅ frame-src 'self' https://challenges.cloudflare.com → Fixes Turnstile iframe issue while blocking other iframes.
✅ object-src 'none' → Blocks old Flash-based attacks.
✅ base-uri 'self' → Prevents attackers from hijacking <base> URLs.
✅ form-action 'self' → Ensures forms only submit to your own site (prevents phishing).

Known Attack Prevention:

Injection Attacks (SQLi, NoSQLi, Command Injection)
-No database used
-No execution of shell commands so command injection is not possible

Cross-Site Scripting (XSS)
-No user input is being stored and rendered dynamically in your front-end.
-API does not return unescaped user-controlled content that could be injected into the front-end.
-No includion of third-party scripts dynamically that could be exploited.
Prevention:
-Added the following in Nginx config:
add_header X-XSS-Protection "1; mode=block";
-NEED TO ADD CSP add_header

Cross-Site Request Forgery (CSRF)
-API endpoints require a Turnstile token, which an attacker cannot generate from another site.
-Transactions require wallet signatures, so even if someone tricked a user into clicking something, it wouldn’t authorize a malicious transaction.
Prevention:
-Cloudflare's "SameSite" cookie policy also prevents CSRF.

Cross-Origin Resource Sharing (CORS) Misconfiguration
-CORS is correctly configured— API only accepts requests from your domain.
-Requests from external domains are blocked (we tested this, and they failed with CORS errors).
Prevention:
-Added the following in Nginx config:
'Access-Control-Allow-Origin': 'https://tifdao.com',
'Access-Control-Allow-Methods': 'POST',
'Access-Control-Allow-Headers': 'Content-Type',

Clickjacking
-Site cannot be embedded in an iframe, which prevents clickjacking attacks.
Prevention:
-Added the following to Nginx config:
add_header X-Frame-Options "DENY";

Server-Side Request Forgery (SSRF)
-Backend does not make user-controlled requests to internal services or external APIs.
-Turnstile validation does not call user-supplied URLs.
Prevention:
-Cloudflare Worker does NOT allow custom URLs in requests.

Distributed Denial of Service (DDoS)
-Cloudflare protects your site against large-scale DDoS.
-Turnstile prevents bots from abusing my API.
Prevention:
-Enable Cloudflare “I’m Under Attack Mode” if you ever get targeted.

Man-in-the-Middle (MITM) Attacks
-Site enforces HTTPS through Cloudflare SSL.
-Sensitive requests are signed using ECDSA, making data tampering impossible.
Protection:
-Added the following in Nginx config:
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload";

Brute-Force & Enumeration Attacks
-Turnstile prevents automated attacks.
-No user login system exists, so no password brute-forcing is possible.

Supply Chain Attacks (NPM package vulnerabilities)
-Updated ethers.js v5.8.0, which removed security vulnerabilities.
-Need to update other packages

Wallet Security
-MetaMask must sign all transactions, which prevents unauthorized transactions.
-Backend never stores private keys.
